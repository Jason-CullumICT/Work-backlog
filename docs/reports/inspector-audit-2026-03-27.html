3. Executive Summary

  1. Zero authentication/authorization on the entire API surface (RED-001). Every endpoint — including state-transition actions, data creation, deletion, and the Prometheus metrics endpoint — is accessible to any network-reachable client without credentials. This is the single most impactful finding and cascades into 5+ related findings.
  2. Persistence layer never loads from disk on restart (CHAOS-002). loadFromFile() is exported but never called in app.ts. Every backend restart loses all work items. The file-based store also uses synchronous I/O (PERF-003), blocking the event loop on every write.
  3. Prometheus metric label cardinality explosion (PERF-001). The HTTP histogram middleware uses raw request paths (including UUIDs) as the route label, creating unbounded time series that will OOM Prometheus at any meaningful scale.
  4. Missing error handling in route handlers (CHAOS-001, CHAOS-009). Intake webhook routes and CRUD routes lack try/catch, meaning store errors will crash the Express process rather than returning error responses.
  5. No input validation or size limits (RED-005, RED-008). Free-text fields accept unlimited-length strings, pagination accepts unbounded limit values, and the overrideRoute parameter bypasses the entire assessment pipeline without authorization.

9. P1 Findings

RED-001 — All API endpoints have zero authentication NEW red-teamer

File: Source/Backend/src/app.ts

Exploit: Any network-reachable client can call every endpoint — CRUD, workflow actions, dashboard, metrics — without credentials.

Impact: Full data read/write/delete. The entire workflow system is compromised.

Recommendation: Implement auth middleware (JWT/session) before all /api/* routes. Add RBAC for state-transition endpoints.

RED-002 — Intake webhooks have no origin verification NEW red-teamer

File: Source/Backend/src/routes/intake.ts:26-89

Exploit: Any client can POST to /api/intake/zendesk and inject arbitrary work items into the pipeline.

Impact: Workflow injection — fake work items proceed through the full pipeline.

Recommendation: Validate Zendesk X-Zendesk-Webhook-Signature-256 HMAC. Require bearer token for /automated.

RED-003 — Prometheus /metrics publicly accessible NEW red-teamer

File: Source/Backend/src/app.ts:44-47

Exploit: GET /metrics exposes route structure, team names, throughput, and Node.js internals.

Impact: Information disclosure enabling fingerprinting and targeted attacks.

Recommendation: Expose on a separate internal port or add bearer-token auth.

PERF-001 — High-cardinality route label in HTTP histogram NEW performance-profiler

File: Source/Backend/src/app.ts:22-29, Source/Backend/src/metrics.ts:10-16

Exploit: req.path includes concrete UUIDs as route label values. Each unique ID = new time series × 6 buckets.

Impact: Prometheus OOM at ~10,000 unique paths. Production-threatening.

Recommendation: Use req.route?.path (Express pattern) or 'unknown' for 404s. Never use req.path as a label.

PERF-003 — Synchronous file I/O on every write operation NEW performance-profiler

File: Source/Backend/src/store/workItemStore.ts:20-34

Exploit: fs.writeFileSync blocks the event loop on every create/update/delete. All concurrent requests are queued.

Impact: p95 latency spikes beyond 500ms budget under any concurrency. Fundamental Node.js model violation.

Recommendation: Replace with fs.promises.writeFile. Add write debouncing.

CHAOS-001 — Intake routes have no try/catch NEW chaos-monkey

File: Source/Backend/src/routes/intake.ts:44, 77

Exploit: Store errors (validation, persistence) throw unhandled exceptions that crash the Express process.

Impact: All webhook intake endpoints become unavailable. Incoming events are lost.

Recommendation: Wrap route bodies in try/catch. Return 500 on unexpected errors.

CHAOS-002 — loadFromFile is never called — store starts empty after restart NEW chaos-monkey

File: Source/Backend/src/app.ts, Source/Backend/src/store/workItemStore.ts:37

Exploit: loadFromFile() exists but is never called in app.ts. All data vanishes on restart.

Impact: Complete data loss from user's perspective after any restart. File on disk is intact but unused.

Recommendation: Call loadFromFile() in app.ts before the server starts listening.

🛑 Block Deployment
